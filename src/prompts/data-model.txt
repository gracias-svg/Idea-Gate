## 2. Core Entities

### 2.1 User Entity

**Purpose:** Store user account information

**Fields:**
- `id` (UUID, Primary Key) - Unique user identifier
- `email` (String, Unique, Not Null) - User email address
- `passwordHash` (String, Not Null) - Hashed password (bcrypt)
- `name` (String, Nullable) - User display name
- `role` (Enum: 'user', 'admin') - User role
- `emailVerified` (Boolean, Default: false) - Email verification status
- `createdAt` (Timestamp, Default: NOW()) - Account creation time
- `updatedAt` (Timestamp, Auto-update) - Last update time

**Indexes:**
- Primary: `id`
- Unique: `email`
- Index: `createdAt` (for sorting)

**Constraints:**
- Email must be valid format
- Password hash must be present
- Email must be unique

## 4. Database Schema (SQL)

```sql
-- Users table
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  name VARCHAR(255),
  role VARCHAR(20) DEFAULT 'user',
  email_verified BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_created_at ON users(created_at);

-- [Main Resource] table
CREATE TABLE [resources] (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  title VARCHAR(255) NOT NULL,
  description TEXT,
  status VARCHAR(50) DEFAULT 'draft',
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_resources_user_id ON resources(user_id);
CREATE INDEX idx_resources_status ON resources(status);
CREATE INDEX idx_resources_created_at ON resources(created_at);
```

## 6. Query Patterns

### 6.1 Common Queries

**Get user by email:**
```sql
SELECT * FROM users WHERE email = ?;
```

**Get user's resources:**
```sql
SELECT * FROM resources
WHERE user_id = ?
ORDER BY created_at DESC
LIMIT 20 OFFSET ?;
```

**Search resources:**
```sql
SELECT * FROM resources
WHERE title ILIKE ? OR description ILIKE ?
ORDER BY created_at DESC;
```

### 6.2 Performance Considerations
- Use indexes for frequently queried fields
- Add composite indexes for common query combinations
- Consider pagination for large result sets
- Use prepared statements to prevent SQL injection
